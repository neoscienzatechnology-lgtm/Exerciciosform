01 - Formulário de contato

Nome, email e mensagem.

Dica: Use type="email" e o atributo required para validação básica; coloque labels claros e um textarea para a mensagem.

02 - Formulário de login

Esqueci a senha

Dica: Para login, adicione autocomplete="username" e autocomplete="current-password"; considere autofocus no primeiro campo.

03 - Formulário de cadastro

Dica: Verifique a confirmação de senha via JS e mostre feedback quando as senhas não coincidirem; valide força mínima com pattern se desejar.

04 - Select com optgroups

Dica: Use um <option disabled selected> como placeholder e agrupe opções com <optgroup> para melhor organização.

05 - Upload de arquivo

Dica: Use o atributo accept="image/*" ou outros tipos; exiba o nome/preview do arquivo com JS (FileReader).

06 - Checkboxes (preferências)

Dica: Para enviar várias opções, use nomes de campo como name="opcoes[]"; agrupe visualmente com grid ou flex.

07 - Radio buttons (fieldset)

Gênero

Dica: Use <fieldset> e <legend> para acessibilidade; todos os radios devem compartilhar o mesmo name.

08 - Textarea autoexpandível (estrutura)

Dica: Para autoexpandir, ajuste a altura via JS no evento input (setar style.height = 'auto' e depois style.height = scrollHeight + 'px').

09 - Telefone (input type=tel)

Dica: Use inputmode="tel" e maxlength para melhorar a experiência em celulares; combine com um pattern para validação.

10 - Barra de pesquisa

Dica: Para buscas, use method="get" se quiser ver a query na URL; adicione role="search" para acessibilidade.

11 - Campos inline (horizontal)

Dica: Quando alinhar campos horizontalmente, mantenha labels visíveis ou use sr-only para oferecer contexto aos leitores de tela.

12 - Grupo de endereços (grid)

Dica: Use atributos de autocomplete como autocomplete="postal-code" e autocomplete="street-address" para acelerar preenchimento automático.

13 - Disabled e readonly

Dica: Campos disabled não são enviados no formulário; readonly é enviado mas não editável. Estilize para ficar claro ao usuário.

14 - Campos obrigatórios

Dica: Use required em campos essenciais e combine com aria-invalid e mensagens de ajuda para acessibilidade.

15 - Datalist (sugestões)

Dica: <datalist> sugere valores, mas não valida — combine com pattern se precisar validar o formato.

16 - Slider (range)

Dica: Mostre o valor atual do range em texto próximo ao slider; defina min, max e passos claros (step).

17 - Seletor de cor

Dica: Use o seletor de cor para UX rápida; exiba o valor HEX ao lado para referencia e compatibilidade com CSS.

18 - Multi-step (estrutura)

Campos da etapa 2

Revisão

Dica: Para multi-step real, mantenha o estado em JS (ou em múltiplos forms) e adicione uma barra de progresso para orientar o usuário.

19 - Input-group (ícone + input)

📞

Dica: Marque ícones como aria-hidden="true" se forem decorativos; sempre forneça um label/placeholder claro.

20 - Validação por pattern (CPF)

Dica: Use inputmode="numeric" e um title claro; para alta confiabilidade, valide CPF no servidor.

21 - Agendamento (datetime-local)

Dica: A aparência de datetime-local varia entre navegadores; considere normalizar datas no backend e avisar sobre fuso horário.

22 - Preview de imagem (estrutura)

Preview

Dica: Use FileReader para criar um dataURL e mostrar a imagem no container; valide o tamanho e tipo do arquivo.

23 - Mostrar/Ocultar senha (estrutura)

Dica: Ao implementar o toggle, altere o atributo type entre password e text e atualize aria-pressed no botão.

24 - Floating label (estrutura)

Dica: Use as classes peer e utilitários como peer-focus do Tailwind para implementar floating labels sem JS.

25 - Formulário em cartão

Dica: Um cartão deve ter padding generoso e uma sombra suave (shadow-lg); mantenha o foco dos inputs visível.

26 - Aceitar termos antes do submit

Dica: Torne o checkbox obrigatório (required) ou desabilite o botão até que seja marcado para garantir conformidade com termos.

27 - Botão reset

Dica: O botão reset restaura os valores iniciais: use-o com cuidado se preferir limpar apenas campos específicos via JS.

28 - Formulário acessível

Dica: Vincule labels com for e id, use aria-describedby para mensagens de ajuda e garanta estados de foco visíveis.

29 - Layout responsivo (mobile-first)

Dica: Siga a estratégia mobile-first: estilize para telas pequenas e depois adicione md:/lg: modificadores para desktop.

30 - Envio via Fetch (estrutura)

(Implemente o JS para interceptar o submit e enviar com fetch.)

Dica: No JS, faça event.preventDefault(), use new FormData(form) e envie com fetch; trate erros e mostre feedback ao usuário.
